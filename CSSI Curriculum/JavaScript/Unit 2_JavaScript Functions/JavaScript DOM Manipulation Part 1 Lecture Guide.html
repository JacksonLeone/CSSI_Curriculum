JavaScript DOM Manipulation Part 1 Lecture Guide

Sequence

  1. Manipulating HTML
  2. Selecting Elements
  3. Adding Event Listeners
  4. Changing Element Text
  5. Changing Element Style

Learning objectives

  • Select elements using document.querySelector().
  • Add event listeners to selected elements using .addEventListener().
  • Respond to events using both named and anonymous callback functions.
  • Modify content of an HTML element using .innerHTML.
  • Modify inline styling of an HTML element using .style

Manipulating HTML

So far, our JavaScript programs have been pretty one-dimensional - we’re just typing stuff into the console. We could write powerful functions, but we’d still need visitors to our webpage to open up the console before they could use any of our functions.

It’s time to get into the real fun - using JavaScript to manipulate parts of the page!

To do so, we will need to do three things:

  • select the element from the page
  • add an event listener to the selected element
  • change something about the element when the event happens

Selecting Elements

The first step is to select the element(s) that you want to interact with. This is done by using document.querySelector() - which is a function that accepts one argument: the string representing the selector. The selector is the same as you’d use in CSS, so for example:

  • document.querySelector('#main') will grab the element with an ID of main.
  • document.querySelector('button') will grab the first button element.
  • document.querySelector('.red') will grab the first element with a class of red.

Notice how document.querySelector() will only select the FIRST of the selector - if there are multiple tags that fit the selector, only the first one will be grabbed. Try to use selectors that are as specific as necessary so as to not mistakenly select the wrong element. It’s worth knowing that for ID selectors in particular, there should only one of element with a given ID. However, if HTML for a given page has mistakenly been written with duplicate IDs, this will only grab the first.

NOTE: If the situation arises where you want to deal with multiple elements at the same time, you will need tools that are covered in the JavaScript DOM Manipulation 2 lesson.

This function returns a value that represents the element - so we can save it to a variable.

For this example, create a button with an ID of like and a text of Like:

1

<button id="like">Like</button>

To select this in JavaScript, we might do something like the following:

1

const likeButton = document.querySelector('#like');

Note that we should always be using const for variables that we know will not change.

Adding Event Listeners

The next part is to add an event listener - this will “listen” (wait) for an event to happen. In particular, we will add an event listener to likeButton, so it will wait for an event to happen to that specific button. We do this with .addEventListener()

The addEventListener function takes two arguments:

  • a string representing the type of event to listen (wait) for
  • a function representing the callback, i.e. the function to be called when the event happens

Here is documentation for events that can happen on the page. We’re going to add an event listener for the click event, and we’re going to use an anonymous function for the callback.

Here’s how it will look:

1
2
3
4
5
6

const likeButton = document.querySelector('#like');
likeButton.addEventListener('click', (event) => {
  // run this code when the button is clicked
  // let's start with a console.log statement:
  console.log('Like button clicked!');
});

Inside the curly braces is the code that will be run when the button is clicked. Let’s do two things when the button is clicked:

  • make the text Liked! instead of Like
  • make the background color lightgreen

Changing Element Text

HTML elements have a property called innerHTML that can be used two ways. Let’s start by logging likeButton.innerHTML to the console - note no parentheses, because it is not a function!

You should see Like appear in the console. So that’s one - we can use innerHTML to retrieve the text in the element.

However, we can also treat it like a variable, and assign a value to it. In this case, we want to assign the value 'Liked!' to it, so we can do:

1

likeButton.innerHTML = 'Liked!';

Now, we only want to do this when the button is clicked, so let’s insert that into the anonymous function representing the callback of addEventListener:

1
2
3
4
5

const likeButton = document.querySelector('#like');
likeButton.addEventListener('click', (event) => {
  console.log('Like button clicked!');
  likeButton.innerHTML = 'Liked!';
});

Now refresh your page, click the button, and the text should change to Liked!.

Once this works, we can remove or comment out the console.log('Like button clicked!'); statement if we choose. We no longer need it to verify that the button is working - we now have visible proof on our page that the event is firing.

Changing Element Style

Now let’s change the background to green. This is done similarly to .innerHTML, except with .style, plus some:

We can discover what values specific CSS properties have by referencing element.style.propertyName. For example, if we wanted to know the color and font-size of our button, we could do the following:

1
2
3

const likeButton = document.querySelector('#like');
console.log(likeButton.style.color);
console.log(likeButton.style.fontSize);

The properties are by default empty, and will only print if the default value is overridden.

Note that we use the formatting fontSize instead of font-size in JavaScript - the first style is called camelCase (the recommended naming convention for JavaScript), while the second is called spinal-case or kebab-case (the standard for most of the CSS we will write).

Note: You need to use camelCase when modifying styles in JavaScript; trying to use spinal-case will result in a syntax error.

We can also set the value for particular properties. To change the background color of the Like button when it is clicked (in addition to everything we’ve done so far), we would do something like:

1
2
3
4
5

const likeButton = document.querySelector('#like');
likeButton.addEventListener('click', (event) => {
  likeButton.innerHTML = 'Liked!';
  likeButton.style.backgroundColor = 'lightgreen';
});

Try it out, and the color should change! If it’s instant and you want the color to change gradually, in your style.css sheet, give it the following:

1
2
3

#like {
  transition: 400ms all;
}

This will make any visual changes apply over a 400 millisecond time period.

Mini-challenges
  1. Rewrite this JavaScript so that the text after you click like is more personalized.
  2. Try modifying at least two other style properties aside from backgroundColor.
  3. Add another button with a different ID to your HTML and then try to repeat the process:
    • Write a querySelector in your JavaScript.
    • Add an even listener to the selected element.
    • Confirm the event is firing with a console.log() statement.
    • Code out some changes to the button using the .innerHTML and .style properties.
  4. What happens if you replace the assignment operator = after .innerHTML with the incrementor +=?
