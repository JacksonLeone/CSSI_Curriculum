JavaScript Conditionals Quiz

Question 1

What is a boolean?

  1. A statement that is either true or false
  2. A block of code that is executed conditionally
  3. The efficiency of your program
  4. A fruit

Answer

A boolean is a statement that is either true or false. They are used within conditionals.

Question 2

What is wrong with the following JavaScript code snippet? Assume x is defined.

1
2
3
4
5
6
7

if (x > 0) {
  console.log('x is positive!');
} else {
  console.log('x is negative!');
} else if (x === 0) {
  console.log('x is neither positive nor negative!');
}
  1. You cannot have a condition following an else if
  2. The order should be if-else if-else
  3. It should be split up into three separate if statements

Answer

The order should be if-else if-else - in other words, it must start with an if and end with an else (if there is one). Having an else if after an else would throw a syntax error.

Question 3

True or false: With an if-else statement, one or both of the code blocks will be run.

  1. True
  2. False

Answer

False - in an if-else statement (even if there are else ifs in the series), only one of the code blocks will run. Once it executes one block of code, it will skip the rest.

Question 4

The following code will check to see if, given an age, the user is old enough to drive and old enough to vote (assume the driving age is 16, since it varies by state). However, when entering a value of 20 for age, it does not appropriately print out both statements. Why not?

1
2
3
4
5

if (age > 16) {
  console.log('You can drive!');
} else if (age > 18) {
  console.log('You can vote!');
}
  1. It is missing an else statement
  2. The check for age > 18 should come first
  3. Since the second conditional is an else if, it will not run if the first conditional evaluates to be true.

Answer

Since the second conditional is an else if, it will not run if the first conditional evaluates to be true. If you wanted to check each condition separately, they should each be in their own separate if statements.

Question 5

The following code will, given a grade from 0-100, convert that grade to a letter grade (A, B, C, D, F). However, entering 65, 100, and 84 each show the grade as a D. Why is this the case?

1
2
3
4
5
6
7
8
9

if (grade >= 60) {
  console.log('Grade is a D');
} else if (grade >= 70) {
  console.log('Grade is a C');
} else if (grade >= 80) {
  console.log('Grade is a B');
} else if (grade >= 90) {
  console.log('Grade is a A');
}
  1. The math is incorrect
  2. The conditionals should be more specific (and have an upper bound)
  3. The order is incorrect
  4. The wrong boolean comparator is used

Answer

Either the order is incorrect, or the conditionals should be more specific. More specific conditionals (e.g. using if (grade >= 60 && grade < 70) for a D would make it work), but ordering it so that the most specific conditional comes first would also work (grade >= 80 also encompasses any grade where grade >= 90, so the latter is more specific).

Question 9 (Challenge)

You are trying to withdraw money from a bank. The bank will not allow you to withdraw more money than you have in your account. Which of the following correctly implements this limit?

Assume there are two variables defined:

  • withdrawAmount that represents the amount you are trying to withdraw
  • accountBalance that represents how much you have in your account

Algorithm 1:

1
2
3
4
5

if (withdrawAmount >= accountBalance) {
  console.log('OK!');
} else {
  console.log('Denied!');
}

Algorithm 2:

1
2
3
4
5

if (withdrawAmount <= accountBalance) {
  console.log('OK!');
} else {
  console.log('Denied!');
}

Algorithm 3:

1
2
3
4
5

if (withdrawAmount === accountBalance) {
  console.log('OK!');
} else {
  console.log('Denied!');
}
  1. Algorithm 1
  2. Algorithm 2
  3. Algorithm 3
  4. None of them
  5. All of them

Answer

Algorithm 2 correctly implements the limit, because it only prints OK! when withdrawAmount is less or equal to accountBalance. If it were false, then you would be trying to withdraw more money than is available.

Question 10 (Challenge)

You’re implementing a login system and you want each user to pick a secure password - in this case, a password with 10 or more characters which does not contain the user’s name. Which of the following algorithms will correctly implement this password validation?

Assume there are two variables and one function defined:

  • name - the user’s name
  • password - the proposed password to check
  • notValid() - this function should be called if the password is not valid. If it is valid, then it should not be called at all.

The function a.includes(b) will check if b is contained anywhere within string a.

Algorithm 1:

1
2
3
4
5

if (password.length < 10) {
  notValid();
} else if (password.includes(name) === true) {
  notValid();
}

Algorithm 2:

1
2
3
4
5
6

if (password.length < 10) {
  notValid();
}
if (password.includes(name) === true) {
  notValid();
}

Algorithm 3:

1
2
3
4
5

if ((password.length >= 10) && (password.includes(name) === false) {
  console.log('OK');
} else {
  notValid();
}
  1. Algorithm 1
  2. Algorithm 2
  3. Algorithm 3
  4. None of them
  5. All of them

Answer

All of them are valid implementations. We basically want notValid() to be called if either of the two conditions (password.length >= 10 and password.includes(name)) are true. The first two are equivalent since it will always perform every check. Algorithm 3 is taking the opposite approach and marking as invalid all passwords unless they match a check for both criteria.
