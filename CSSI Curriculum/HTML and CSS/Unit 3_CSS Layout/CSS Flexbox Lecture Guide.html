CSS Flexbox Lecture Guide

Framing

Our content is starting to look good thanks to some CSS styling and Material Design principles, but it still all shows up vertically with each element stacked right below the previous one - we can create sidebars or listicles, but we will want additional tools when we need content to show up in any sort of horizontal arrangement.

Context & purpose

Adding content, picking colors, and fonts is only half the battle to creating an eye-catching website. One of the most difficult parts is laying out your content in a way that is appealing. With layout, you can decide how much space each of your elements will take up on your page and where on the page they should be placed. There are many ways to use CSS to lay out your site, but the way that we’re going to do it is by using the CSS flexible box (flexbox) model.

Learning objectives

Students will be able to

  • Use CSS flexbox as a way to lay out their page.
  • Assign properties to parent elements to achieve desired layout goals.
  • Assign properties to child elements to achieve desired layout goals.

Wireframing

A wireframe is a visual representation or sketch of a webpage. Why would it be important to know what you want your website to look like before you start coding it? This may seem obvious, but you’d be amazed at how many first-time developers just jump in and start throwing code on an html file.

Flexbox

Parent and Child: Basic flex.

The divs with a class of “row” will be acting as rows; they will be parent elements. The divs with a class of “item” will be contained inside the rows, so they’ll become child elements.

The most critical pieces of information for the parent elements has to do with how it will display its contents. We’ll give it the css property display: flex;. Flex has a LOT of different uses, but at this most basic level, assigning display: flex; to a parent element means the items inside it will have some amount of flexibility in terms of their size and positioning.

Mini-challenges
  1. Open your HTML page in the browser. Notice how the items are displayed currently.
  2. Add the property and value display: flex; to the .row ruleset.
  3. Refresh the page in your browser and see what changed.

Flexbox has turned our rows into actual rows, which is a really good starting point, but this is only the beginning. We have a lot more control over those rows than you might imagine.

The next few sections explore specific properties (mostly for the parent elements with a class of "row") that allow us more control over these elements.

Flex Direction

The first property worth experimenting with is the flex-direction property. The four values you’re likely to see are row, column, row-reverse, and column-reverse.

To see these in action and figure out exactly how they’ll interact with your code, try assigning one value to each of the specific rows. Do this one at a time so that you have a chance to modify each one and examine it closely.

Align Items

Let’s now experiment with another property: align-items. The four values you’re most likely to see associated with this property are stretch, flex-start, flex-end, and center. You may also see baseline, but you aren’t likely to need that right away.

To start, test out the stretch value:

1
2

#row1 {
  align-items: stretch;
}

If you noticed no change, that’s because stretch is the default behavior of the align-items property. So even without a CSS rule, the content was already stretched.

Justify Content

The third flex property for the parent is the justify-content property, and its arguably the most commonly used and the most powerful. The values you’ll see most often associated to it can be broken up into two categories:

Values that cluster content together:

  • flex-start
  • flex-end
  • center

Values that space content apart:

  • space-between
  • space-around
  • space-evenly

Let’s examine the differences between the clustered-content values first. As always, do these one at a time, and notice the way that each property changes that row’s behavior.

Combining Properties

Each of these properties can have a calming effect on its own, but you can achieve some of the most striking visual results by combining two or more properties in a single row. Try this ruleset, for example:

1
2
3
4

#row3 {
  justify-content: space-evenly;
  align-items: center;
}

At this point, you should start experimenting with some combinations and examining the results. Feel free to be creative, or try the mini-challenges below.

Mini-challenges
  • Have all the items clustered together in the exact center of a row.
  • Have all the items clustered in the bottom right corner of a row.
  • Spread the items out as far apart as possible, and align them to the top of the row.
  • Cluster the items together in the bottom center of the row.
  • STRETCH: Cluster the Swift and HTML divs together, but leave the rest of the items spaced apart.
  • STRETCH: Have the second row display as a column, and center the items within that column.

You may encounter some trouble with the last two challenges. Click below for some solutions and some additional information about each of them.

Stretch 1 solution:

To make the Swift and HTML divs cluster together, the best way is to modify the HTML even further.

1
2
3
4

9

<div class="row" id="row3">
      <div class="subrow">
        <div class="item" id="swift">Swift</div>
        <div class="item" id="hyper">HTML</div>
      </div>
      <div class="item" id="aframe">A-Frame</div>
      <div class="item" id="css">CSS</div>
      <div class="item" id="java">Java</div>
    </div>

We’ve grouped them both in a div with a class of "subrow", so now the div with an ID of "row3" only has four direct children (the div with a class of "subrow", and the A-Frame, CSS, and Java divs). It just so happens that the first child has two children of its own.

That technically completes the mini-challenge, but you’ll notice that JUST changing the HTML will have made our Swift and HTML divs cluster together in a vertical stack. That actually makes sense, since the default behavior of almost all HTML is to stack vertically, but we actually already know how to fix that:

1
2
3

.subrow {
  display: flex;
}

We’ve created a flexbox WITHIN a flexbox - that might seem complicated right now, but it’s actually a really common structure, and can be used to create all sorts of interesting effects.

Stretch 2 solution:

You may have realized that the first step in this process was this:

1
2
3

#row2 {
  flex-direction: column;
}

And previously, to center things horizontally, we’ve used the rule justify-content: center;, but if you tried it, it likely didn’t work.

The rule you actually need is the align-items: center;. In context, it looks like this:

1
2
3
4

#row2 {
  flex-direction: column;
  align-items: center;
}

The trick here is that at first, it seems like justify-content is about horizontal positioning, and align-items is about vertical positioning, but that’s only true when the flex-direction is set to row or row-reverse.

In fact, justify-content controls positioning parallel to the flex-direction, and align-items controls positioning perpendicular to the flex-direction.

With that said these naming conventions and explanations aren’t completely intuitive, and even many full-time front-end developers often just try a few combinations, and then if they’re still stuck, they look back at a reference like the flexbox cheat sheet to double check their thinking.

Don’t put pressure on yourself to memorize this; just remember that flex-direction can change how the other properties behave.

.item {
  flex-grow: 1;

The fact that flexbox items can be sized to take up all the extra space available makes it a lot easier to design sites that look great on small, medium, and large screens - this is a property that may come in handy later on.

The flex-grow property is just one of several flexbox properties that can be given to child elements, but most other child element properties tend to be most useful when you want one of the child elements to behave differently from its siblings.

Since those cases are a bit rarer, you should feel free to dig into additional child properties in various forms of documentation. One particularly useful resource is the flexbox cheat sheet, which has a list of all the most common flexbox properties for both parent and child elements, as well as visual illustrations of what each property is likely to do.

Best Practices Summary

Resources

In this lesson, you learned how to assign parent and child elements the appropriate flexbox properties and values, but the most common use of flexbox is almost always to center a div (or a cluster of divs) on a page. That process is almost always the same, so here’s a summary of how to make that happen:

  1. In the HTML, wrap the element (or elements) you wish to center in a parent element (usually a div), and give that element a unique class or ID.
  2. In the CSS, assign that parent element the display: flex;, justify-content: center;, and align-items: center; properties.

With that said, no matter how experienced you are, you’ll want to reference the flexbox cheat sheet whenever you’re looking to implement a specific behavior.
