Framing

Let’s take an example from before, when we used a dictionary to represent a pet.

pet = {
  "name": "Doggo",
  "animal": "dog",
  "species": "labrador",
  "age": 5
}

What if we added more to this pet? What if we wanted to also give it some behaviors as well? For example, let’s say we wanted to also celebrate the pet’s birthday (increasing its age by 1). How could we do that?

Using the skills we already have, we could write a function that takes in a pet dictionary argument, mutates it, but a more widely-used solution in Python is to use objects.

Object-Oriented Programming

Object-oriented programming (or OOP) is a model organized around “objects”. Programmers using OOP define both the type of data for an object, and the way that object will behave.

Before we create an object, we need to have a class. A class can be thought of as a template or blueprint for an object - an outline that describes what data and behaviors an object will have. Here’s a simple Pet object that we’ll be expanding as we learn more about OOP.

1
2
3
4

class Pet(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age

A note on naming - it’s standard to start class names with capital letters, and for names with multiple words, like “Bank Account”, to use CamelCase, where the first letter of every word is capitalized (i.e. BankAccount) - no underscores.

Pet is the name of the class, which describes the object that we are defining. After the class name, we put object in parentheses. We’ll talk more about this later – in general terms, it’s a way of saying “every instance of this class is an object.” Watch out: it’s not the same as a function argument, even though it borrows the same syntax.

The __init__() method is the method that gets called when we instantiate, or create an instance, of the Pet class (i.e. when we create a Pet object using the word Pet with parentheses after it). This is called the constructor. self is a required argument that references the object itself, and the name and age are additional data we are sending into the method.

The __init__() method is called automatically when we create an instance of an object - you won’t ever call it explicitly.

Notice how there isn’t any actual pet information in the class definition - that’s because it’s an outline, a blueprint - to actually create a Pet, we need to provide values:

1
2
3
4
5

# Initialize a new instance of pet, with a name of "Fido" and an age of 3.
my_pet = Pet("Fido", 3)

# This should print `My pet Fido is 3 years old`.
print("My pet %s is %s years old" % (my_pet.name, my_pet.age))

Note that even though we never called the __init__() method, the code from that method (storing the name and age as parts of the self) must have run in order for our print statement to run without error. Remember that __init__() always runs automatically as a part of object initialization, which was done with the line my_pet = Pet("Fido", 3).

  • Classes are blueprints for objects
  • Create an object by providing the class with values
  • Access an object’s data by using object.property_name

Classes and Properties

The data within an object is called its properties (or state). Let’s add a few more to our Pet object. Try adding the following properties: animal (string), is_hungry (boolean), mood (string).

# defining the blueprint for a Pet
class Pet(object):
    def __init__(self, name, age, animal):
        self.name = name
        self.age = age
        self.animal = animal
        self.is_hungry = False
        self.mood = "happy"

# create a new pet using the Pet blueprint
my_pet = Pet("Fido", 3, "dog")

# printing some values out to test
print("%s is a %s %s" % (my_pet.name, my_pet.mood, my_pet.animal))

This should print Fido is a happy dog

In your attempt, you may have put all five properties as parameters in the init method. That works really well. However, you may notice that in the above example, we’ve done something clever: we don’t require arguments for every property. In our __init__() definition, we only ask for name, age, and animal, then we supply some default values for the other states.

That means that when we initialize a new pet, we don’t have to specify its hunger or its mood - those are built into our __init__() method. In this case, we’re assuming that our pets start off happy and well-fed, because we’re good caretakers.

Each of these new properties describes the state of the object - is the pet hungry? What is the pet’s mood? How old is the pet?

Remember that an object is data as well as behaviors. The properties are the data that describe the object - but we still need to learn about its behaviors.

Recap

  • Properties are the data/state that belong to the object
  • You can set default values for properties

Instance Methods

# defining the blueprint for a Pet
class Pet(object):
    def __init__(self, name, age, animal):
        self.name = name
        self.age = age
        self.animal = animal
        self.is_hungry = False
        self.mood = "happy"

    # the pet is eating
    def eat(self):
        print("> %s is eating..." % self.name)
        if self.is_hungry: # This is equivalent to `self.is_hungry == True`
            self.is_hungry = False
        else:  # If the first test was false, and the pet wasn't hungry...
            print("> %s may have eaten too much." % self.name)
            self.mood = "lethargic"

# defining the blueprint for a Pet
class Pet(object):
    def __init__(self, name, age, animal):
        self.name = name
        self.age = age
        self.animal = animal
        self.is_hungry = False
        self.mood = "happy"

    # the pet is eating
    def eat(self):
        print("> %s is eating..." % self.name)
        if self.is_hungry:
            self.mood = "happy"
            self.is_hungry = False
        else:  # eating when already full
            print("> %s may have eaten too much." % self.name)
            self.mood = "lethargic"

    # the pet is playing
    def play(self):
        print("> %s is playing..." % self.name)
        self.is_hungry = True
        if self.mood == "happy":
            self.mood = "excited"
        elif self.mood == "excited":
            self.mood = "tired"
        print("> %s is feeling %s." % (self.name, self.mood))

Recap

  • Define a class’s behaviors by using instance methods
  • Instance methods can change an object’s properties/state
  • Objects can make your code significantly more readable, and can reduce the confusion of using a nested data structure.
