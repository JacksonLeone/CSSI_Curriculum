Context

Functions in Python, as in other languages, are blocks of code that can be called upon to be run. As in other languages, functions are created with a function definition and then run or executed with a function call. While our JavaScript functions were used primarily to control and manipulate the DOM (the HTML document-object model), Python functions are usually used for more behind-the-scenes work, which makes sense, as Python is typically thought of as a back-end scripting language.

Python Functions In Action

Defining Functions

Define a function with the def keyword, followed by the name, followed by a colon. The code “inside” the function should be indented, similar to how code as part of an if/for/while is also indented. For example:

1
2

def say_hello():
    print("Hello World!")

Note: def say_hello(): is called the function header or function signature.

Calling/Running Functions

Call a function simply by using its name, followed by parentheses.

1

say_hello()

(prints Hello World!)

Functions with Arguments

Functions can have an input - they can take in data through arguments. Arguments (input) are saved through variables defined in the function header.

1
2

def greet(name):
    print("Hello, %s!" % name)

When you call a function with arguments, you have to include those arguments (the actual values) in the parentheses of the function call.

1

greet("Harry Potter")

(prints Hello, Harry Potter!)

You can have a function with multiple arguments - the values are assigned in order:

1
2
3
4

def greet(first_name, last_name):
    print("Hello %s! %s is such a lovely last name!" % (first_name, last_name))

greet("Ada", "Lovelace")

(prints Hello, Ada! Lovelace is such a lovely last name!)

Functions with Return Values

Functions can also return values by using the return keyword. All function calls always represent the function’s return value (which is None if nothing is returned).

def count_vowels(word):

    count = 0
    for letter in words.lower():
        if letter in vowels:
            count = count + 1
    return count

vowels = count_vowels('awesome')
print("Vowels counted: %s" % vowels)

Prints the same thing, but notice that now we have a variable vowels that keeps the value returned - we can use this later on if we need to.

Tips & Tricks

  • Any time you find yourself tempted to repeat a multi-line block of code, consider packaging that code in a function.
  • Name your functions so that you remember what they do.
    • If a function will have a return value, consider naming the function after the return value itself, e.g. the name cost_with_tax(item) suggests that it’s designed to return a numerical value.
    • If a function will not have a return value, consider naming the function after the task it performs, e.g. reset_game() likely resets a lot of values inside of a program, but may not return any data.
  • Whitespace (and specifically indentation) is how Python groups lines of code together. Indent your code consistently throughout your code to make it clear which lines of code are “inside” function definitions, and which lines of code are not.
  • Include control flow (if/for/while) if necessary, but remember to indent this content one level further than the rest of the function.

Best Practices Summary

  • Functions are defined with the keyword def.
  • The function header includes the function name, parentheses, a list of parameters (if applicable), and a colon.
  • Indent every part of a function after its header.
  • Calling (running) a function requires the parentheses, even if the function doesn’t require arguments.
  • Name parameters descriptively, to make your code more readable for yourself and for other developers you collaborate with.
    • For example, change_password(old_password, new_password) is a lot more precise and easier to understand than change_password(str1, str2).
