Context

Python is a scripting language that, at its core, has a lot of the same capabilities as JavaScript, but is used in vastly different ways. While JavaScript is often used to control user-facing visual output (and is therefore generally thought of as a front-end scripting language), Python is generally used to process data behind the scenes (and is therefore generally thought of as a back-end scripting language).

Python In Action

Variables

Variables are defined in python simply by assigning a value to a variable. Variable assignment is completed with an equal sign - assigning the value on the right hand side of the equal sign to the variable on the left.

1
2
3

name = "John"
print("Hi there..")
print(name)

Printing

Similar to Javascript’s console.log, python uses the function print().

Any line starting with a # is considered a comment and not executed. (also, any text after a # is considered a comment as well, even if there is code before it.)

1
2
3

print("Hello world!")
# this is a comment!  It is ignored and not run.
print("Bye world!")  # this is also a comment. But "bye world" is still printed!

User Input

Python uses the raw_input() function to pause execution to allow the user to type something in. The result is typically saved to a variable. The prompt that the user sees would go inside the parentheses, inside of quotes. See the example:

1
2
3

name = raw_input("Enter your name: ")
print("Hi there..")
print(name)

Note that a colon and a space is put in the prompt, as the user’s input would begin after the space on the same line.

Operators

Operators on Numbers (integers, decimals [floats])

Addition, subtraction, multiplication, and division (+-*/ respectively) do exactly what you would expect. There are also operators:

  • % Modulo - remainder (a % b is the remainder of a / b)

= // Floor division - division, then taking the floor of the result (cuts off decimal places, always rounds down)

Operators on Strings (text)

You can add strings together (note that it doesn’t add a space in automatically!):

1

print("Howdy" + "there")  # prints 'howdythere'

You can also multiply strings by an integer:

1
2

print("na " * 16)
print("BATMAN!")

Data Types and Type Conversions

Python, like JavaScript, represents data as strings or numbers. It also includes lists, the equivalent of JavaScript arrays, and dictionaries, the equivalent of JS Objects. We will learn how to use both in the next lessons.

We can tell what datatype something is by using the type() function.

For example:

1
2

user_input = raw_input("Enter anything: ")
print("You entered a " + type(user_input))

We can convert that string to another datatype by using the following conversion functions:

  • int(x) converts x (the value inside) to an integer (whole number)
  • float(x) converts x (the value inside) to a float (decimal number)

Strings cannot be mixed with integers, so we have to convert the number (integer) back to a String to print it by using the str() conversion function:

1
2
3
4

num1 = int(raw_input("Enter number #1: "))
num2 = int(raw_input("Enter number #2: "))
total = num1 + num2
print("The sum is " + str(total))

Conditionals

An if/elif/else statement in python might look like the following:

1
2
3
4
5
6
7

num = int(raw_input("Enter a number: "))
if num > 0:
    print("That's a positive number!")
elif num < 0:
    print("That's a negative number!")
else:
    print("Zero is neither positive nor negative")

Some things to note:

  • When comparing values, Python uses == and != which are more strict than == and != in Javascript. For example in Python 5 != '5', while in JavaScript5 == ‘5’(JavaScript has the stricter operators===and!==` that behave similarly to Python’s operators)
  • The logical operators in Python are and, or and not, as opposed to JavaScript’s &&, || and !
  • elif is equivalent to else if in JavaScript
  • In python, no parentheses are required around the condition (unlike JS), but are accepted
  • Instead of using {} to indicate code blocks, code blocks are instead determined by the level of indentation. Indenting forwards means a code blocks is opening; indenting backwards (left) means a code block is closing.

Loops

while loops

1
2
3
4
5

# counts 1 to 5
x = 1
while x <= 5:
    print(x)
    x = x + 1

for loops

In Python, for loops always loop through an iterable, such as a list (equivalent of a JavaScript array) or a string (which can be seen as an array of characters).

1
2
3

string = "hello there"
for letter in string:
    print(letter.upper())

This prints each letter in capitals, one per line, by looping through the string.

In python, the range() function will generate a list of numbers that you can then iterate over in a for loop.

Examples:

  • range(5) will give you a list of numbers [0,1,2,3,4]
  • range(5,10) will give you a [5,6,7,8,9]
  • range(5,1,-1) will give you [5,4,3,2]

We can use this in a for loop:

1
2

for i in range(5):
    print(i)

String Formatting

Consider the following code:

1
2
3
4
5
6
7
8
9

my_name = "Bob"
friend1 = "Alice"
friend2 = "John"
friend3 = "Mallory"

print(
  "My name is " + my_name + " and my friends are " +
  friend1 + ", " + friend2 + ", " + friend3 + "."
)

This prints:

1

My name is Bob and my friends are Alice, John, and Mallory.

We can make this easier by using String formatting - check out this alternative:

1
2
3
4

print(
  "My name is %s and my friends are %s, %s, and %s" %
  (my_name, friend1, friend2, friend3)
)

The %s is a placeholder for a value. After the string (quotes end), put a % followed by the values to insert, in order, in parentheses. If it’s just one value, you can omit the parentheses. Example:

1
2

my_name = "Bob"
print("My name is %s." % my_name)

You also don’t have to convert values to use them in string formatting, so it’s very useful when mixing types:

1
2

age = 12
print("I'm %s years old." % age)

Tips & Tricks

  • Sometimes an error in Python is difficult to trace without lots of print() statements to debug your code. Add these throughout to see if your code is running in the way you expect it to.
  • When adding print() statements, consider adding two of them (one with a string, one with a variable) so that the output will be easier to decipher in the console.
    • First print a hard-coded string to show what’s being printed next: print("User 1's password:")
    • Then print the variable print(user_1_password)
  • Conditional statements have lots of parts, so they can be challenging to debug. If a statement like if password == user_guess1: isn’t working correctly, break it into three parts (the left side, the right side, and the whole boolean expression) to debug it:
    • print(password)
    • print(user_guess1)
    • print(password == user_guess1)
  • One of the most common errors for beginners is mixing up the assignment operator = (used to set variables) with the boolean expression operator == (used to check whether two values are the same).
    • When using an if statement, always use the == operator.
  • Indentation matters in Python. It must be consistent throughout your script.

Best Practices Summary

  • Store data in variables that reflect what the variable references (e.g. age instead of x or username instead of string1).
  • Name variables using snake_case capitalization.
  • Include spaces on either side of operators in Python (e.g. age = 3 + 1, where = and + are both operators).
  • Colons for conditional statements or loops should be immediately adjacent to the preceding value (e.g. if age > 18: where there is no additional space between 18 and :).
