Learning objectives

  • Master in Python the same principles covered in JavaScript.
  • Store data in Python variables.
  • Store data submitted by a user through the console in a variable.
  • Use mathematical operators to manipulate numerical data.
  • Convert between Python data types.
  • Use logical operators to write if-elif-else control flow.
  • Write while and for loops in Python.
  • Print strings from user data using concatenation and string formatting.
  • Write custom Python function definitions and call those functions.

Most of what’s covered in this lesson mirrors some of the concepts covered in the JavaScript unit. Although Python cannot manipulate the DOM in the same way JavaScript can, the syntax of Python often feels a little more approachable than that of JavaScript.

Read on for examples of how to execute some of the most fundamental scripting tasks in Python.

Getting Started

If you’re doing this work in the native OSX terminal, you’ll use the workflow to create and run scripts in terminal. Remember that Python scripts end in .py, so create a Python file with a command like touch myfile.py. Check to confirm that the file is indeed in your current working directory with ls. Execute a python script with the command python myfile.py.

Right now, the command python myfile.py won’t do anything. That’s because even though we’ve created a file, we still need to write some code in that file to be printed out to the terminal when it is run.

First Program

Similar to JavaScript’s console.log(), Python uses the function print().

Any line starting with a # is considered a comment and not executed.\ Any text after a # is considered a comment as well, even if there is code before it.

1
2
3

print("Hello world!")
# this is a comment!  It is ignored and not run.
print("Bye world!")  # this is also a comment. "Bye world" is still printed!

Variables

In Javascript, variables have to be first be declared with var or let/const.

Variables are defined in Python simply by assigning a value to a variable. Variable assignment is completed with an equal sign - assigning the value on the right hand side of the equal sign to the variable on the left.

1
2
3

name = "John"
print("Hi there,")
print(name)

User Input

Python uses the raw_input() function to pause execution to allow the user to type something in. The result is typically saved to a variable. The prompt that the user sees would go inside the parentheses, inside of quotes. See the example:

1
2
3

name = raw_input("Enter your name: ")
print("Hi there, ")
print(name)

Note that a colon and a space is included in the prompt, as the user’s input begins immediately after the string on the same line. Run this program now to see how it works.

Operators

Operators on Numbers (integers, decimals [floats])

Addition, subtraction, multiplication, and division (+, -, *, and / respectively) do almost exactly what you would expect. There are also a few less familiar (but equally useful) operators - for now we’ll draw attention to just two of those:

** - the power/exponent operator: the expression 2^5 (or two to the fifth power) is written as 2 ** 5. % - Modulo - the remainder operator: a % b is the remainder of a / b.

Data Types and Type Conversions

Python, like JavaScript, represents data as strings or numbers. It also includes lists, the equivalent of JavaScript arrays, and dictionaries, the equivalent of JS Objects. We will learn how to use both in the next lessons.

We can tell what datatype something is by using the type() function.

For example:

1
2

user_input = raw_input("Enter anything: ")
print("You entered a " + type(user_input))

Run this a few times and, and try entering numbers. It should always tell you that you’re entering a String. That’s because raw_input() always takes the user’s input as a string.

We can convert that string to another datatype by using the following conversion functions:

  • int(x) converts x (the value inside) to an integer (whole number)
  • float(x) converts x (the value inside) to a float (decimal number)

Write a program that takes in two integers and prints the sum. You should end up with something like this:

1

num1 = int(raw_input("Enter number #1: "))
num2 = int(raw_input("Enter number #2: "))
print(num1 + num2)

Conditionals

An if, else-if, else statement in Python might look like the following:

1
2
3
4
5
6
7

num = int(raw_input("Enter a number: "))
if num > 0:
    print("That's a positive number!")
elif num < 0: # Note that we use the one-word elision "elif", not "else if"
    print("That's a negative number!")
else:
    print("Zero is neither positive nor negative")

Click to see the equivalent in JavaScript:

var num = parseInt(prompt("Enter a number"));
if (num > 0) {
  console.log("That's a positive number!");
} else if (num < 0) {
  console.log("That's a negative number!");
} else {
  console.log("Zero is neither positive nor negative");
}

Some critical things to know about conditional operations in Python:

  • elif is equivalent to the else if of JavaScript.
  • The logical operators in Python are and, or and not, as opposed to JavaScript’s &&, || and !.
  • In Python, parentheses are not required around the condition.
    • Parentheses are allowed, and will be executed exactly the same as in JavaScript, but typically only come into play if a developer is trying to adhere to a style guide’s character limit requirements. In general, Google’s guidance for parentheses in Python is that they should be used sparingly.
  • Instead of using {} to indicate code blocks, code blocks are instead determined by the level of indentation of each line of code.
    • In JavaScript, indentation is not required to run code, but it is generally expected and does help make our code more readable.
    • In Python, indentation is required. It is the only way you can clarify when a code block is beginning, continuing, or ending. Indenting forwards means a code blocks is opening; indenting backwards (left) means a code block is closing.
  • When comparing values, Python uses ==, which is equivalent to JavaScript’s === operator. Click below for more detail on the variations between equality checks in these two languages:

While it wasn’t explicitly covered in the JavaScript unit, JavaScript also has a == equality comparison operator, which you may have seen as you checked online references to solve bugs. However, JavaScript’s == isn’t the same as Python’s == operator.

Python’s == is more strict than the == in Javascript. JavaScript allows the comparison of a string and an integer 5 == "5" to return true since the numerical values match, even though the types do not. Python comparisons are not so flexible. In Python, 5 == "5" would return False.

In JavaScript, the type-converting == comparison also allows comparison between null and undefined to be true. This is the most useful application of JavaScript’s == comparison operator, and according to Google’s style guide, is the only occasion where the use of == is accepted. In all other situations, developers are encouraged to use the === strict comparison.

An example of indentation at work:

age = 20
if age > 18:
    print("You can buy lottery tickets!")
    # This next if statement is still part of our initial if code block.
    if age > 25:
        print("You can also rent a car!")
    # Since we've indented backwards, we're exiting the `if x > 25` block.
    print("And you can get your own credit card!")
# Since we've indented backwards again, we're exiting the `if x > 18` block.
print("End program.")
# Code not indented at all is sometimes called "top-level code".

Loops

for loops - strings

Python for loops work a little differently than JavaScript for loops. Python for loops always loop through an iterable, such as a list (equivalent of a JavaScript array) or a string (which can be seen as an array or list of characters).

1
2
3

string = "hello there"
for letter in string:
    print(letter.upper())

This prints each letter in capitals, one per line, by looping through the string.

NOTE: The iteration variable letter is never explicitly defined before starting the for-in loop, but the loop still runs as expected. Click to learn why the loop runs without needing more information.

Bear in mind that a string is called a “string” because it a is string of characters. Python is built on the assumption that you might want to do something for every character in that string.

That means that this loop will run one time for each character in the string, and the iteration variable will point to each letter in turn: the first time the loop runs, letter will be "h", the second time, letter will be "e", the third time it will be "l", and so on until each character in "hello there" has taken a turn as the value of letter.

The word letter itself is arbitrary, and could be replaced with another word like char or c. You can use whatever term you want for the iteration variable, but it’s best practice to have it somewhat clearly represent the type of information that will be stored there each time the for loop runs, which in the case of strings is a letter or character.

Click for a JavaScript comparison.

In JavaScript, we would mostly write a for loop over a range of numbers. For example, to count from 0 to 4:

for (let i=0; i<5; i++) {
  console.log(i);
}

Based on this example, you could figure out out how to iterate over the characters in a JavaScript string. Take a few minutes and try it. Is it harder or easier than iterating over the characters in a Python string?

for loops - lists

For loops can also iterate over a Python list - this is a data structure that isn’t covered until the next lesson, so there’s no need to learn it now. However, if you’re interested in learning more ways to use for loops in Python, click to see how looping over a list can work.

1
2
3

books = ["The Hobbit", "The Hunger Games", "The House on Mango Street", "Native Son"]
for book in books:
  print("I really enjoyed reading " + book + " this year.")

NOTE: Just as above, the singular word book is used as our iteration variable, but is never explicitly defined. In the same way that a string is a collection of characters, a list is a collection of things - in this case, a collection of book titles stored as strings.

So even though book was never explicitly defined, it becomes our iteration variable as part of the for-in syntax.

You’ll notice that the list was called books but the iteration variable doesn’t have to be the singular version of the array name, book. You could just as easily use for title in books: or for x in books:. You could even use for pineapple in book: but the ridiculousness of that variable name should be obvious. You’ll notice that the more abstract the iteration variable becomes, the harder it becomes to read (and ultimately debug) our own code.

For that reason, even though it’s not a hard and fast rule, the convention of naming our iteration variable something that can very specifically stand in for each value contained in the list. In this case, book is probably the best choice.

for loops - range()

In Python, you may wish to do something a specific number of times. The range() function will generate a list of numbers that you can then iterate over in a for loop.

Examples:

  • range(5) will give you a list of numbers [0,1,2,3,4]
  • range(5,10) will give you a [5,6,7,8,9]
  • range(5,1,-1) will give you [5,4,3,2]

We can use this in a for loop to do something a specific number of times:

1
2

for i in range(5):
    print(i)

This same goal - doing something a specific number of times - can be accomplished in JavaScript as well, though the syntax looks a little more complicated. Click to see the same example in JavaScript.

1
2
3

for (i = 0; i < 5; i++) {
  console.log(i);
}

while loops

Looping in Python can also be accomplished with while loops, which execute as over and over as long as a given condition is true.

1
2
3
4
5

# This loop counts 1 to 5.
x = 1
while x <= 5:
    print(x)
    x = x + 1 # The variable x is incremented by 1 each time.

Python while loops are a powerful way to repeat code in ways that for loops cannot. For example, you can use the expression while True: to repeat code endlessly.

1
2
3

# This loop will run forever, because the condition is literally always True.
while True:
    print("Loop running.")

String Formatting

To understand when this next concept might be most useful, try writing code that stores your name and three friends’ names in variables, and then prints out a single sentence explaining who you are and who your friends are. Click below to see an example when you’re ready.

1
2
3
4
5
6
7
8
9

my_name = "Bob"
friend1 = "Alice"
friend2 = "John"
friend3 = "Mallory"

print(
  "My name is " + my_name + " and my friends are " +
  friend1 + ", " + friend2 + ", " + friend3 + "."
)

This prints:

1

My name is Bob and my friends are Alice, John, and Mallory.

Even though the result is simple, printing lots of variables in a single line can get really messy with constant closing quotes and plus signs.

We can make this type of task easier by using String formatting - check out this alternative:

1
2
3
4

print(
  "My name is %s and my friends are %s, %s, and %s" %
  (my_name, friend1, friend2, friend3)
)

The %s is a placeholder for a value. After the string (quotes end), put a % followed by the values to insert, in order, in parentheses. If it’s just one value, you can omit the parentheses. For example:

1
2

my_name = "Bob"
print("My name is %s" % my_name)

Mixing types

To see another example of when string formatting can be most helpful, try writing code that stores your name as a string in a variable called name, and your age as an integer in a variable called age. Then print out a sentence indicating who you are and how old you are. Try doing this twice: once using concatenation and once using string formatting. Click below when you’re ready or if you get stuck to see a solution:

You may have gotten a TypeError: cannot concatenate 'str' and 'int' objects message on your attempt at concatenation. If you handled it on your own, great job! But if not, it’s a pretty simple fix.

1
2
3
4
5
6
7
8

name = "Marina"
age = 19

# Using concatenation, you must convert the integer to a string:
print("My name is " + name + " and I'm " + str(age) + " years old.")

# Using string formatting, the type is converted automatically:
print("My name is %s and I'm %s years old." % (name, age))

Functions

Functions in Python are defined using the def keyword.

1
2

def greetAgent():
    print("Bond. James Bond.")

You call (run) functions by using their name, followed by parentheses.

1
2

greetAgent()
# This calls the function we defined and prints `Bond. James Bond.`

Click to see how this same function would be defined and called in JavaScript:

1
2
3
4
5
6
7

// Function definition:
const greetAgent = () => {
  console.log("Bond. James Bond.")
};

// Function call:
greetAgent();

The parentheses are required, even if they are empty. You can pass data to the function through the parentheses - these are called arguments.

1
2
3
4
5
6

# the greetAgent function expects two pieces of data to be passed to it:
#   A value that gets saved into `first_name`, and
#   a value that gets saved into `last_name`.

def greetAgent(first_name, last_name):
    print("%s. %s %s." % (last_name, first_name, last_name))

Call it with:

1
2
3
4
5

# These values must be provided in the order they were defined.
# So we know that "James" will be the `first_name`, and "Bond" the `last_name`
greetAgent("James", "Bond")
# As before, the beauty here is that we can reuse the function with other data.
greetAgent("Beyonce", "Knowles")

Click to see the same function defined in JavaScript:

1
2
3
4
5
6
7
8

// Function definition:
const greetAgent = (first_name, last_name) => {
  console.log(`${last_name}. ${first_name} ${last_name}.`)
};

// Function call:
greetAgent("James", "Bond");
greetAgent("Beyonce", "Knowles");

Note that we pass 2 values because the function expects to be passed 2 values. Passing in the wrong number of values is is the most common error you’re likely to encounter when you first start working with your own custom functions, so reference your function definition if you’re struggling to see why a function call is throwing an error.

Functions can also return values to where it was called by using the return keyword.

1
2

def createAgentGreeting(first_name, last_name):
    return "%s. %s %s." % (last_name, first_name, last_name)

Now if you call createAgentGreeting("James","Bond") - nothing happens! That’s because return doesn’t actually print anything. The function call itself (in this case, createAgentGreeting("James","Bond")) is replaced with the return value.

We can save it in a variable or we can print the result directly if we don’t expect to need the data later.

1
2
3
4
5
6
7

# Method 1: store the return of the function in a variable.
greeting = createAgentGreeting("John", "Smith")
# Print the contents of that variable - we can use it later in our program.
print(greeting)

# Method 2: print the return value directly, without saving the data.
print(createAgentGreeting("Bob", "The Builder"))

Return values are especially useful when you might not want to print the result, or you need to perform some kind of calculation and do something with the result later on. Printing is useful when you’re hunting down a bug, working with a really small dataset, or building a command line application where a user will interact with your product directly in their terminal, but there may be times when excessive printing clutters up your console in an unhelpful way.

Return is a much more versatile result for a function for a number of reasons that will become increasingly clear the more code you write. In general, you’ll find it better to return values from your functions, store those values when you write the function call, and then print those variables if you need to check the return value of a function.
