Learning objectives

  • Store multiple pieces of data in a single variable using a Python list.
  • Access and modify elements in a list.
  • Use the built-in list methods .append(), .insert(), .remove(), and len().
  • Iterate over elements in a list.
  • Iterate over elements in a list by index.
  • Check whether a certain element is or is not in a list.

Accessing Elements

Now try the following code:

1
2
3
4

students = ["Alice", "Javi", "Damien"]

print("The three students are:")
print(students[0] + "\n" + student[1] + "\n" + student[2])

We have things to unpack from the previous example:

  • Each item in a list, called an element of the list, can be represented by its index.
  • The index is an element’s position in the list.
  • Indexing starts from zero. The first element of a list has an index of zero. The last element of a list has an index of the length minus one. Remember this - you will need it later.

You can reference an element in a list by using its index in square brackets next to the list name.

List Methods

For right now, the four most important list methods you’ll need to know include:

  • .append()
  • .insert()
  • .remove()
  • len() NOTE: there’s no . in front of len(). That’s because len() is not specifically a list method; it is a function that works on lists, and it’s called slightly differently.

Instead of demonstrating how to use each of these, see if you can figure out how to use each by tackling the mini-challenges below. When you get stuck on any of them, you’ll want to reference some Python docs on List Methods. You may be able to guess how to use some of the methods listed above, but it’s almost always better to look at examples in documentation than to try to guess syntax all on your own.

List Iteration By Index

Now consider the following example - we’ve added a few family members).

1
2
3
4
5

smith_siblings = ["Emily", "Monique", "Giovanni", "Brianna", "Javi"]
for index in range(len(smith_siblings)):
    smith_siblings[index] = smith_siblings[index] + " Smith"

print(smith_siblings)

Recall that for i in range(5) will loop through the numbers [0,1,2,3,4].

Consider also that in this example, len(smith_siblings) will evaluate to 5.

In a list with 5 elements, 0,1,2,3,4 represent all the indexes in that list! So we can use this type of for loop to loop through each index of the array, instead of the elements themselves.

This allows us to use any number of list methods that require the index, such as insert().

Click to see a more complex example with conditionals inside the for loop.

1
2
3
4
5
6
7
8

names = ["Rickon", "Bran", "Arya", "Sansa", "Jon", "Robb"]
for i in range(len(names)):
    if names[i] == "Jon":
        names[i] = names[i] + " Snow"
    else:
        names[i] = names[i] + " Stark"

print(names)

Note that the constant of iteration doesn’t have to be named i or index, but use of those names is fairly common because they help us keep track of what type of information (in this case, a number) is contained in that variable.

Note that you can also use the same style of abbreviated reassignment that we’ve used for numerical values (we’ve called it “incrementation”) with strings as well. here’s another way to achieve the exact same result as above:

    else:
        names[i] += " Stark"

Slicing (Optional)

One of the most powerful ways Python can interact with a list (and with any iterable) is through slices.

Take a slice (a section or selection) from a list with square brackets, and follow the syntax my_list[start:end:step]. Each of these values helps determine which entries in the list will be included in a slice.

  • The value used for start will be the index of the first item in the slice. If you do not provide a starting value, the default value is 0, and your slice will start at the beginning of the list.
  • The value used for end is not included, so if you provide a value of 4, the slice will include up to the item with an index of 3. If you do not provide an ending value, the default behavior is to continue until the end of the list.
  • The default value for step is 1, meaning “include every single item”. You could get every other item with a step of 2, every third item with a step of 3.

The most common implementations of slices often only provide ONE of these three values. In fact, step is important infrequently enough that you can also use the syntax [start:end] without step altogether. Below you’ll see examples of both syntaxes.

1
2
3
4

names = ["Rickon", "Bran", "Arya", "Sansa", "Jon", "Robb"]

# Slice starting at index 2:
names[2::] # => ["Arya", "Sansa", "Jon", "Robb"]
# This could also be written as names[2:].

# Slice until (not including) index 4:
names[:4:] # => ["Rickon", "Bran", "Arya", "Sansa"]
# This could also be written as names[:4]

# Slice starting at index 2 and until index 4:
names[2:4:] # => ["Arya", "Sansa"]
# This could also be written as names[2:4]

# Slice including only every second name:
names[::2] # => ["Rickon", "Arya", "Jon"]
# This cannot be written without both colons, since we're using the third value.

The value of step is actually most commonly used in a very specific situation. When we want to get a slice of a list reversed, we use the value -1. Below are two examples of how that might work.

1
2
3
4
5
6
7

names = ["Rickon", "Bran", "Arya", "Sansa", "Jon", "Robb"]

# Slice of the entire list from beginning to end, except reversed:
names[::-1] # => ["Robb", "Jon", "Sansa", "Arya", "Bran", "Rickon"]

# Slice of reversed list, starting at idx 4 and continuing backwards to idx 2:
names[4:2:-1] # => ["Jon", "Sansa"]

NOTE: Slicing subsets of a list in reverse is a little trickier than normal slices - the value for start is, in this case, greater than the value for end. It takes some practice, and even experienced developers will check their work on this sort of thing with some debug statements.
