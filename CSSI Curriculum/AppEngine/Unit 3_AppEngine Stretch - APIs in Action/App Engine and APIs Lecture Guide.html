Open APIs

Section purpose

In this section, you will be reintroduced to APIs. You will then read some TrivaDB Documentation and finally try to build out a trivia app all on your own.

Here’s what you should know:

  • APIs can grab distinct pieces of data.
  • Reading the documentation helps identify endpoints - in fact, it’s almost the only way.
  • Most APIs require a key to be included in the query string, but some are open
    • we’ll try to focus on the open ones.

Go to the TrivaDB documentation and use the helper to explore the different endpoints.

Once you feel good about endpoints, getting info from them inside your app is pretty simple:

  • urlfetch.fetch gets the response from a url (file-like structure).
  • .content() gets the inside of the response.
  • json.loads() converts the response string to JSON.
  • Parse (read) the response just like a regular Python dictionary.

Below is some sample code that makes a call to the TriviaDB database asking for 10 easy multiple-choice questions. Those questions come back as a JSON (JavaScript Object Notation) object. The app then shows the first question contained in that JSON to the user.

The commented code and pseudocode here is broken out into four distinct steps, including an initial self.response.write() to verify that the app is responding to requests correctly. As you walk through adding each step to the app, test out the code iteratively, first trying Step 0, then Step 1, then Step 2, and finally parsing the JSON in Step 3.

#main.py
import webapp2
import random
#libraries for APIs
from google.appengine.api import urlfetch
import json


class MainPage(webapp2.RequestHandler):
    def get(self):
        trivia_endpoint_url = "https://opentdb.com/api.php?amount=1&category=10&difficulty=easy&type=multiple"
        trivia_response= urlfetch.fetch(trivia_endpoint_url).content
        trivia_as_json = json.loads(trivia_response)
        first_result = trivia_as_json['results'][0]
        trivia_question = first_result['question']
        self.response.write(trivia_question)


app = webapp2.WSGIApplication([
    ('/', MainPage),
], debug=True)

#main.py
import webapp2
import random
#libraries for APIs
from google.appengine.api import urlfetch
import json


class MainPage(webapp2.RequestHandler):
    def get(self):
        trivia_endpoint_url = "https://opentdb.com/api.php?amount=1&category=10&difficulty=easy&type=multiple"
        trivia_response= urlfetch.fetch(trivia_endpoint_url).content
        trivia_as_json = json.loads(trivia_response)
        first_result = trivia_as_json['results'][0]
        trivia_question = first_result['question']
        correct_answer = first_result['correct_answer']
        incorrect_answers = first_result['incorrect_answers']

        #create an array of all 4 answer options
        all_answer_options = [correct_answer]
        for answer in incorrect_answers:
            all_answer_options.append(answer)
        #reorder answer options
        random.shuffle(all_answer_options)
        #render questions on screen
        self.response.write("<h2>" + trivia_question + "</h2>")
        self.response.write(all_answer_options)



app = webapp2.WSGIApplication([
    ('/', MainPage),
], debug=True)

The code above builds HTML inline, as part of the main.py file’s responsibilities. But we know that we could instead pass the question data through to a Jinja3 template, and that would be much better practice. Refer back to the templating lesson for a refresher walkthrough on how to render python variables as part of a template.

Challenges

Build an app using the questions and answers from the TrivaDB API.

  • Start by building an app that shows a question, and then shows the answers on a button click.
  • Add a form for the user to select a level of difficulty or category.
  • Add logic to check for the correct answers. Note that working with True/False questions will be easier to check than multiple choice answers.

NOTE: These aren’t mini-challenges. Each of these items might take between 30-45 minutes - it isn’t necessary to complete all of them to consider your code successful.

APIs with Authentication

Section purpose

This section will introduce you to two types of API authentication: keys and Oauth. You will not build a full app. Instead, you will practice successfully getting a response from the two different types of APIs.

There are two major types of APIs which require authentication:

  • Key-based, where your credentials are passed as part of a query string.
  • OAuth, which uses a client to make a request with user credentials.

This example shows how to make a request with the first type, a key-based API. You’ll want to start by reading through the Giphy documentation - APIs change all the time, so what worked a month ago may not work now. Always check the documentation first.

Next, you’ll need to create an account and find your key on the Giphy Developer Dashboard.

Once you have your key, you’ll be adding that key to the query string.

#main.py
import webapp2
import random
from google.appengine.api import urlfetch
import json


class MainPage(webapp2.RequestHandler):
    def get(self):
        giphy_key = "0f3f7dc74bc64817b3d3b547083f9b8e"
        endpoint_url = "http://api.giphy.com/v1/gifs/search?q=baby+otter&limit=10&api_key="+ giphy_key
        giphy_response = urlfetch.fetch(endpoint_url).content
        response_as_json = json.loads(giphy_response)
        random_index = random.randint(0, 9)
        gif_image = response_as_json['data'][random_index]['images']['original']['url']
        self.response.write(
          "<html><body><img src='{}'/></body></html>".format(gif_image))



app = webapp2.WSGIApplication([
    ('/', MainPage),
], debug=True)

Twilio is a fun last API to try out - it lets you send SMS messages, MMS messages with images, and create some other very cool automated integration with phones. It uses OAuth instead of a token to authenticate.

As before, start by reading through the (Twilio SMS documentation)[https://www.twilio.com/docs/quickstart/python/sms]. One of the most critical steps is to install Twilio’s Python library. Since this is a third-party library, there are a few more steps than normal to make the library work when the app is deployed.

Create a new folder called lib to hold this (and any other) library, and also create a file called appengine_config.py to list the libraries to be used in deployment:

1
2

mkdir lib
touch appengine_config.py

Now install the Twilio library and add it to a local lib folder:

1

pip install twilio -t lib/

You’ll also need to add Twilio to your config file, so that the Twilio library is also loaded in when your app is deployed online (and not just when you are running a devserver).

In appengine_config.py:

1
2
3

from google.appengine.ext import vendor

vendor.add('lib')

Next, create your credentials on the Twilio Developer Dashboard and create a demo number.

Bear in mind that if you hard-code your credentials here and also push your project up to GitHub, then that private information will be publicly available. Twilio lets you send text messages directly to someone’s phone, but some types of messages cost a tiny amount of money (usually less than a cent) to send - if your Twilio were configured with a credit card, and someone else had your API key, they could send texts and YOU would be charged for them.

If you add Twilio credentials (or any API credentials), you might want to store your API credentials in Datastore, or find some other secure way of storing and accessing that data.

#main.py
import webapp2
import os
import random
from google.appengine.api import urlfetch
import json
import twilio # This will cause an error if you didn't `pip install twilio -t lib/` first


class MainPage(webapp2.RequestHandler):
    def get(self):

        # Your Account Sid and Auth Token from twilio.com/user/account
        account_sid = os.environ['TWILIO_SID']
        auth_token = os.environ['TWILIO_TOKEN']
        client = twilio.rest.Client(account_sid, auth_token)

        send_to_phone_num = "+2243921935"
        # Get a trial Twilio number at
        # https://www.twilio.com/console/sms/getting-started/build
        message = client.messages.create(
            send_to_phone_num,
            body="Hello from Chicago",
            from_=os.environ['TWILIO_PHONE'],
            media_url="https://www.publicdomainpictures.net/pictures/170000/velka/chicago-skyline-dusk-view.jpg"
        )

        self.response.write(
          "Your message has been sent to " + send_to_phone_num
          + " with an id of " +  message.sid)



app = webapp2.WSGIApplication([
    ('/', MainPage)
], debug=True)
