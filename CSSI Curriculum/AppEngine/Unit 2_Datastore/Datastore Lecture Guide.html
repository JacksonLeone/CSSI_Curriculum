Databases, Records and Fields

In the database overview pre-reading for this lesson, you learned a lot about databases and how they are used.

The individual Entities in a database will each have several Properties associated to it. In this section, we’ll first create a Kind (a Python class) that describes what type of data is stored in each one of those properties, and then create instances (entities) of that class (kind) to add individual entries to our database.

This terminology of Kind, Entity, and Property are terms unique to Datastore, but the concepts behind them are applicable in other contexts as well - here’s a reference explaining how those terms translate to other data storage tools.

Intro to GAE’s Datastore

In Datastore, each set of properties is represented as a Kind (known more generally as a model). A Kind is a specific class intended to be use with Datastore. Here’s an example:

class Movie(object):
    media_type = 'Movie'

    def __init__(self, mov_title, runtime, user_rating):
        self.title = mov_title
        self.runtime_mins = runtime
        self.rating = user_rating

    def autoplay(self):
        if (self.runtime_mins<=120) and (self.rating>=9.0):
            return "Up Next: " + self.title
        else:
            return "Search for More Movies"

Using Datastore

Now that we’re all up to speed on the type of class we might use as a Kind (or model), it’s time to dive into how to use GAE’s datastore.

The Admin Console

With dev_appserver running, open up the admin tool by pointing your url to localhost:8000.

From there, open up the Interactive Console. The first time you try, you may get an error letting you know that you need to run a more specific command with the --enable_console option. Shut down the app in the terminal, and then use dev_appserver.py app.yaml --enable_console to run it again with the interactive console enabled.

Here’s an example of what a functioning Interactive Console might look like:

The Interactive Console is a tool designed to let developers test out things like defining Kinds, creating individual Entities, storing those Entities in Datastore, and retrieving those Entities from datastore, all in the console. In the same way that your terminal or console allows you to interact directly with your computer, the App Engine Admin Console allows you to interact directly with Datastore.

Movies: From Class to NDB Model

from google.appengine.ext import ndb

class Movie(ndb.Model):

  title = ndb.StringProperty(required=True)
  media_type = ndb.StringProperty(required=True, default="Movie")
  runtime = ndb.IntegerProperty(required=False)
  rating = ndb.FloatProperty(required=False)

  def autoplay(self):
        if (self.runtime<=120) and (self.rating>=9.0):
            return "Up Next: " + self.title
        else:
            return "Search for More Movie Options"

This should look really similar to what we have above. The biggest differences are that the class appears to have an argument ndb.Model, and that the properties are not defined as part of an __init__() method, but are top level and are described as different ndb properties. Just like the request handlers from our App Engine lesson, this is another example of inheritance, where instead of building our Python class from scratch, we’re building off of an existing class (ndb.Model), and using that to simplify the code we have to write.

Create Entities

Since there’s no __init__() method, the next step is to understand how creating an instance of this class (an Entity of this Kind) might be different - it’s almost exactly like creating a regular instance of a regular class, except that the arguments are labeled.

Persist Entities

The last thing we need is a way to save these movies somewhere.

Open up the Datastore Viewer. You’ll notice it’s empty. Now add the code below to our existing movie code:

1
2

movie1.put()
movie2.put()

Go back to the Datastore Viewer. Not only is our information there, but each movie Entity is associated to a unique key and id. This is the core of how we save information. The id is just a number generated by Datastore - the key, however, is a dynamic object that lets us link objects together, and that you can use to retrieve an Entity from Datastore.

But as our users won’t be using an interactive console, we need to modify our meme generator to include this new feature and store the memes not from the interactive console, but directly from our app using the info provided in the HTML form.

NOTE: The drop-down in the Datastore viewer is likely set to Movie right now, since it’s the only Kind you’ve created so far. Once you finish this next part, you’ll need to select “Meme” from the dropdown, and click “list entities” to see the data you’ve sent.

Modify Entities

The beauty of .put() is that it can be used for two purposes - to store new entries and to modify existing entries.

1
2
3
4
5
6

movie1 = Movie(title="Forest Gump", runtime=131, rating=8.3)
movie1.put()

# If a new review comes in, you might want to update the rating.
movie1.rating = 9.1 # This modifies your local copy of the movie object.
movie1.put() # This updates the existing entry in datastore to match the local.

Note that this assumes you still have the local copy of movie1 in a variable somewhere when the updated rating comes through - you can also retrieve a copy of movie1 from Datastore. In fact, retrieving information no longer stored locally is the main reason to use Datastore in the first place. We’ll get to exactly how to do that in the next lesson.

#models.py
from google.appengine.ext import ndb

class Meme(ndb.Model):
  first_line = ndb.StringProperty(required=True)
  second_line = ndb.StringProperty(required=False)
  pic_type = ndb.StringProperty(required=False)

  def get_meme_url(self):
    if self.pic_type == 'old-class':
        url = 'https://upload.wikimedia.org/wikipedia/commons/4/47/StateLibQld_1_100348.jpg'
    elif self.pic_type == 'college-grad':
        url = 'https://upload.wikimedia.org/wikipedia/commons/c/ca/LinusPaulingGraduation1922.jpg'
    elif self.pic_type == 'thinking-ape':
        url = 'https://upload.wikimedia.org/wikipedia/commons/f/ff/Deep_in_thought.jpg'
    elif self.pic_type == 'coding':
        url = 'https://upload.wikimedia.org/wikipedia/commons/b/b9/Typing_computer_screen_reflection.jpg'
    return url
